# End-to-End Test Plan

## Goal

Cover the full import pipeline -- from real filenames through guessit parsing,
mocked TMDB HTTP responses, template rendering, file operations, and DB writes
-- in a single test run. No actual network calls. Deterministic and CI-friendly.

The existing test suite is thorough at the unit level but mocks
`_pipeline.identify()` in service tests, meaning guessit parsing, confidence
scoring, and template rendering never run together. These E2E tests close that
gap.

---

## Test architecture

### Stack under test

```
real filename on disk
  -> scan_media_files (discovery)
  -> parse_filename (guessit)
  -> scan_for_nfo_id (NFO scanner)
  -> compute_hash (OSDB, runs on real bytes)
  -> parse_mediainfo (degrades gracefully, no mediainfo binary in CI)
  -> TMDBSource.search / get_by_id (HTTP mocked with `responses`)
  -> confidence scoring (Jaro-Winkler + year decay)
  -> auto-accept / requires_interaction decision
  -> render_template (destination path)
  -> file ops (copy_verify / move / link / hardlink)
  -> DB write (SQLite in-memory)
  -> companion file handling
```

Everything is real except TMDB HTTP, which is mocked with the `responses`
library to return canned JSON matching what the real API returns.

### Mocking strategy

**Mocked:** TMDB HTTP endpoints via `responses` library. Each test scenario
defines its own TMDB search response JSON so the scoring is exercised against
realistic API payloads.

**Not mocked:** guessit, template engine, file operations, DB, companion
classifier, NFO scanner, OSDB hash, discovery scanner.

**MediaInfo:** Not installed in CI. `parse_mediainfo` returns `{}` gracefully.
Tests do not depend on MediaInfo output.

### File location

All E2E tests go in `tests/test_e2e/`. One file per concern area:

```
tests/test_e2e/
  conftest.py              -- shared fixtures (library dirs, config, TMDB mocks)
  test_movie_import.py     -- movie filename scenarios
  test_tv_import.py        -- TV show filename scenarios
  test_directory_structures.py -- multi-file layouts, nested dirs
  test_companions.py       -- companion file handling through full pipeline
  test_edge_cases.py       -- collisions, re-imports, special chars, failures
```

## Scenario catalog

### A. Movie filenames

Each scenario creates a real file on disk, mocks the expected TMDB response,
runs `import_path`, and asserts on: destination path, DB record fields,
file existence, and summary counts.

## Success and failure criteria

### A test passes when

1. **Destination file exists** at the path predicted by template rendering
2. **Source file state** matches the import mode (exists for copy/link, deleted
   for move)
3. **DB record** (if not --no-db) has correct: path, title, year, media_type,
   tmdb_id, match_source, confidence, mtime, size
4. **Summary counts** (imported, skipped, errors, unmatched) match expectations
5. **Companion files** are at expected locations with correct names
6. **No extra files** created in the library beyond what's expected

### A test fails when

1. File ends up at wrong destination or is missing
2. DB record has wrong field values (especially match_source, confidence)
3. Summary counts are wrong
4. Source file state doesn't match mode (e.g., source deleted in copy mode)
5. Companion files missing, misnamed, or in wrong location
6. Pipeline crashes instead of gracefully handling bad input
7. Sample files are imported (should be excluded)
8. Non-video files are imported

### Invariants (assert in every test)

- `summary["errors"] == 0` (unless testing error scenarios)
- Library directory contains only expected files
- DB record count matches imported count
- No temp files left behind

### File creation helper

```python
def make_video(directory, name, size=1024):
    """Create a dummy video file with given name and size."""
    path = directory / name
    path.parent.mkdir(parents=True, exist_ok=True)
    path.write_bytes(os.urandom(size))
    return path
```

Use `os.urandom` so SHA-256 checksums are realistic and unique.

### NFO creation helper

```python
def make_nfo(directory, name, tmdb_id):
    """Create an NFO file with a TMDB ID."""
    path = directory / name
    path.write_text(f'<movie><tmdbid>{tmdb_id}</tmdbid></movie>')
    return path
```
